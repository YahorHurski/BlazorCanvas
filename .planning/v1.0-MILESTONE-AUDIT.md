## Cross-Phase Integration

**Result: clean — no BLOCKER or WARNING findings.**

Every seam the ROADMAP flagged as high-risk was traced through actual code rather than
inferred from summaries:

| Seam | Risk | Finding |
|------|------|---------|
| Geometry core (P1) → draw (P3) / drag (P4) / render | ROADMAP warned "do not re-implement" | **No re-implementation.** `DrawGesture.Build` composes `Movement.ClampDelta`, `CircleEncoding.ClampDrawRadius/FromCentreRadius` and `Normalisation.Normalise`; `Home.razor:191` calls `Movement.ClampMove` directly; `FigureShape.razor:55` calls `CircleEncoding.ToCentreRadius`. Zero duplicate clamp/normalise logic outside `Geometry/`. |
| Auth (P2) → data isolation (P3/P4) | IDOR spanning three phases | **Sound.** `userId` is read once from the cookie claim (`Home.razor:99`, no DB lookup) and every `FigureStore` query filters `WHERE UserId == userId` (`FigureStore.cs:32,74,93`). The `userId` never originates from client input. |
| Write path (P3/P4) → broadcast (P5) | optimistic broadcast | **Sound.** `draw` fires only after `InsertAsync` returns a DB-assigned id (`Home.razor:288-290`), satisfying D-39. Delete broadcasts only after the DB confirms. |
| Zero-row UPDATE (P4) → delete broadcast (P5) | **highest risk — DATA-03's two halves were built in different phases** | **Correctly wired.** `CommitDragAsync` (`Home.razor:335-344`) removes locally and broadcasts `Delete` in the same method. |
| `move` UPDATE-only (D-40) | resurrection of deleted figures | **Sound.** `ApplyMessage` is UPDATE-only for `move`/`rollback`; unknown id is ignored, never inserted (`Home.razor:486-492`). |
| Sync rules (P5) vs drag machine (P4) | mid-drag filter scope | **Sound.** Echo filter on sender GUID plus D-54's blanket mid-drag discard confirmed in `HandleRemoteMessage` (`Home.razor:437-460`). |

**E2E flows traced (9/9 complete, no breaks):** login → draw all four shapes → select →
drag → delete → F5 reload → second-tab subscribe → cross-tab glide → save-failure rollback.

14 lower-severity items carried open from `01-REVIEW.md` (2026-07-15). None is required by

Two caveats worth recording so they are not mistaken for live debt:

- **WR-01** (nothing rejects an off-canvas figure) is **intentional** — locked D-36 decides
  there is no canvas-bounds CHECK; clamping is C#-side by design.

  D-08 and explicitly not to be "fixed".
- **WR-02** (scoped-per-circuit `AddDbContext`) **appears closed** by 03-02's migration to
  `IDbContextFactory`, but was not re-verified here.

The remaining ~11 items are genuine, low-severity follow-ups. Recommend triaging them into
a backlog rather than blocking the v1.0 close.
