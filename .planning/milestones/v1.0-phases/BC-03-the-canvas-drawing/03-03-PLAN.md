    Create `src/BlazorCanvas/Components/Canvas/FigureShape.razor` (namespace
    `BlazorCanvas.Components.Canvas`, reached automatically from the folder). Add local
    `@using BlazorCanvas.Geometry` — follow `Login.razor`'s pattern of per-page usings rather than
    editing `_Imports.razor`.

    Parameters:
    - `[Parameter, EditorRequired] public FigureType Type { get; set; }`
    - `[Parameter, EditorRequired] public Box Box { get; set; }`
    - `[Parameter] public bool Preview { get; set; }`

    The component emits ONLY bare SVG child elements — one `line`, `rect`, `circle`, or `polygon`
    element selected by `Type` in an `@switch`/`@if` chain. It must never emit a nested SVG root
    element of its own: a nested root establishes a new coordinate system and would break the 1:1
    page-to-canvas mapping this whole phase rests on (D-18). The parent SVG is supplied by
    `Home.razor` in plan 03-04.

    Shape arms:
    - `FigureType.Line` -> a `line` element with `x1="@Box.X1" y1="@Box.Y1" x2="@Box.X2" y2="@Box.Y2"`,
      reading the four values RAW. Do not apply `Math.Min`/`Math.Max` and do not sort the axes: an
      up-and-right line is stored with `Y1 > Y2` and rewriting it here would render the opposite
      diagonal — D-41's landmine, one layer down. A line has no interior, so it carries no fill; its
      2px stroke is the only click target it has (D-32, D-58).
    - `FigureType.Rectangle` -> a `rect` element with `x="@Box.X1" y="@Box.Y1" width="@Box.Width" height="@Box.Height"`.
    - `FigureType.Circle` -> decode with `CircleEncoding.ToCentreRadius(Box)` and emit a `circle`
      element with `cx`, `cy`, `r` from the returned tuple. All three are `int`.
    - `FigureType.Triangle` -> a `polygon` element whose points are apex, base-left, base-right:
      apex `((X1 + X2) / 2.0, Y1)`, base-left `(X1, Y2)`, base-right `(X2, Y2)`. Apex top-centre,
      base along the bottom, always isosceles, always upward (D-21).

    CULTURE LANDMINE, and the reason the apex is the only computed coordinate in this file: the apex
    x is a `double` and is the single place in the app where a fractional coordinate is formatted into
    markup. `CultureInfo.CurrentCulture` on a Blazor Server circuit is the server's OS culture unless
    request localization is configured; on a comma-decimal locale `13.5` formats as `13,5`, and the
    browser silently reparses the points list as a different set of coordinates. Build the points
    string in the `@code` block with a private member that formats through
    `System.Globalization.CultureInfo.InvariantCulture` (for example via `FormattableString.Invariant`),
    and bind it as `points="@ThatMember"`. Do not use integer division to dodge this — that would make
    odd-width triangles non-isosceles, contradicting D-21.

    Style, applied as literal SVG attributes rather than via a scoped CSS file (attributes cannot be
    lost to a CSS-isolation scope mismatch, and this component renders inside a parent's SVG):
    - Every arm: `stroke="#000000" stroke-width="2"` (`figure.stroke`; 2px is deliberate — D-58).
    - `rect`, `circle`, `polygon`: `fill="#FFFFFF"`. THE FILL IS LOAD-BEARING (D-38): SVG does not
      register clicks inside an unfilled shape, and Phase 4's "pressing inside a rectangle grabs it"
      is only true because of it. Never render these as wireframes.
    - When `Preview` is true, add `stroke-opacity="0.7"` and `fill-opacity="0.7"` (03-UI-SPEC:
      the preview reads as "not yet real" without inventing a new colour, which D-14 forbids).

    Do not add a `Selected` parameter, a red stroke, or any click handler — selection is Phase 4
    (FIG-02). Do not create a `FigureShape.razor.css`.
