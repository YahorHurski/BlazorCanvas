### The most valuable restore

## Outstanding Work — MIGR-03 only

Three files remain deleted, all tied to MIGR-03. They were excluded from this pass deliberately: they
need a design decision, not a mechanical rebase.

- `Database/V11/V11MigrationReplayTests.cs` (337 lines)
- `Database/V11/V11MigrationReplayFixture.cs` (243 lines)
- `Database/V11/V11SchemaShapeTests.cs` (349 lines) — partially superseded by
  `V11CutoverTests.AssertFinalPublicCatalogAsync`

The decision the replay test needs:

- **Cheap path.** The standalone `V11DataMigration.RunAsync(dataSource, registry)` overload survived
  unchanged in `Transition/` and still produces `v11.*` while leaving legacy tables intact, so the
  replay test restores nearly verbatim. But that overload is now unreferenced in production — it
  would prove the interim path, not the cutover path users actually get.
- **Correct path.** Drive `V11Cutover.EnsureAsync` and assert against `public.figures`. This requires
  snapshotting legacy rows *before* cutover (cutover drops `public.figures`) and discarding
  `Migration_LeavesTheOldTablesAndConstraintsIntact`, which becomes false by design.

Either way the assertions themselves — 8 tabulated rows with literal expected vertices, old-id `z`
ordering, per-row jsonb style, and bbox recompute — transcribe directly from the existing
`v1.1-pre-rewrite-MANIFEST.md`.

## Process Finding

Phase 11's plan authorised removing "tests whose subject no longer exists". The executing agent
applied that to any test that failed to compile after the legacy model was dropped — which included
tests whose subject was *promoted*, not removed. Phase 11's verification then reported `passed`
against a suite 77% smaller than the one Phase 10 was verified against, because it checked only its
own four requirements and the suite was green.

That the restoration built on the first attempt is the strongest evidence the original deletion was
over-broad: 6 of the 9 wrongly-deleted files needed nothing but a schema rename.

A cheap guard for future milestones: when a phase's plan authorises test deletion, have its
verification compare total test count against the previous phase's recorded count and justify any
drop. Both numbers are already written into the VERIFICATION.md files.
