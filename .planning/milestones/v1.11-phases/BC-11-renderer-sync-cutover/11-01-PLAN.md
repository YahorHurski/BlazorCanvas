<objective>
Prepare the additive v1.11 store for live use without removing the v1.1 safety net. Build one
owner-to-canvas resolver, one startup coordinator, and the DI composition consumed by the renderer
and circuit cutover in Wave 2.
</objective>

<threat_model>
| STRIDE | Threat | Disposition |
|---|---|---|
| Spoofing | A caller supplies another user's predictable canvas UUID. | Mitigated: CanvasRepository derives the id from authenticated owner id; repository operations remain canvas-scoped. |
| Tampering | Reload/startup creates duplicate canvases or replays rows. | Mitigated: deterministic id plus parameterized `ON CONFLICT` and idempotent migration. |
| Repudiation | Bootstrap failure leaves a mixed model silently served. | Mitigated: coordinator fails loudly on invalid catalog state before routes are mapped. |
| Information disclosure | A pooled source bypasses ownership isolation. | Mitigated: pooling carries no user state; every query includes canvas ownership. |
| Denial of service | Every request triggers a global migration. | Mitigated: coordinator runs only at bounded startup; owner ensure is a single idempotent insert. |
| Elevation of privilege | DI creates a raw JSON repository path. | Mitigated: repository remains `ValidatedFigureInput`-only. |
</threat_model>
