<objective>
Execute the one destructive operation only after the app consumes v11 data: promote the migrated
schema into public, remove legacy storage/geometry/testing scaffolding, and rebase the lasting suite
to the final public model.
</objective>

<threat_model>
| STRIDE | Threat | Disposition |
|---|---|---|
| Spoofing | Destructive helper is pointed at an unintended database. | Mitigated: configured source only, catalog identity checks, and guarded scratch database tests. |
| Tampering | Partial DDL loses data or promotes wrong tables. | Mitigated: verify/migrate first, then one transactional lock/drop/ALTER sequence; reject partial states. |
| Repudiation | Restart mistakes failed cutover for completed cutover. | Mitigated: explicit catalog-state probe makes completed run no-op and partial run fatal. |
| Information disclosure | Rebasing removes canvas ownership predicates. | Mitigated: repository predicates stay unchanged and post-promotion cross-owner tests remain. |
| Denial of service | Startup repeats destructive DDL or leaves locks. | Mitigated: bounded transaction, completed-state no-op, rollback on failure. |
| Elevation of privilege | Legacy guard removal opens raw JSON writes. | Mitigated: retain gateway/repository hostile-input test and no-raw-overload assertion. |
</threat_model>
