<objective>
Build the style half of the validation gateway: the `FigureStyle` typed record with the D-66 bounds
(colour whitelist `^#[0-9A-Fa-f]{6}$`, stroke width clamped 0.5-64, opacity clamped 0-1) and the

now the only thing standing between a hostile payload and a rendered attribute.

Create `src/BlazorCanvas/Shapes/FigureStyle.cs`, namespace `BlazorCanvas.Shapes`:

`public sealed record FigureStyle(string Stroke = "#000000", double StrokeWidth = 2, string Fill =
"#FFFFFF", double Opacity = 1)`.

Those four defaults are not arbitrary â€” they are the literal values `FigureShape.razor` renders for a
committed figure today, and Phase 10's migration writes them onto every existing row so nothing
changes on screen (D-66, MODEL-06, MIGR-01). Add a comment saying so.

Add a private static readonly compiled colour regex `^#[0-9A-Fa-f]{6}$`. Use `[GeneratedRegex]` with
a partial method if the C# version in use supports it, otherwise
`new Regex("^#[0-9A-Fa-f]{6}$", RegexOptions.Compiled | RegexOptions.CultureInvariant)`. It must be a
single static instance, never constructed per call. Anchored at both ends, no alternation, no nested
quantifier, no backreference â€” state in a comment that this shape cannot backtrack catastrophically.

Add `public FigureStyle Sanitised()` returning a new record where:

- `Stroke` is kept if it matches the regex, otherwise replaced with `#000000`.
- `Fill` is kept if it matches the regex, otherwise replaced with `#FFFFFF`.
- `StrokeWidth` is replaced with `2` when `double.IsFinite` is false, then clamped into `[0.5, 64]`
  with `Math.Clamp`.
- `Opacity` is replaced with `1` when `double.IsFinite` is false, then clamped into `[0, 1]`.

The finiteness replacement MUST happen before the clamp. `Math.Clamp(double.NaN, 0.5, 64)` returns
NaN, so a NaN that reached the clamp would survive it, be written into `figures.style`, and then be
emitted into an SVG `stroke-width` attribute as the literal text `NaN`. Put that reasoning in a
comment next to the check â€” it is the single non-obvious line in this file.

Also handle a null `Stroke` or `Fill`: a `null` reaching `Regex.IsMatch` throws. Treat null as
non-matching and substitute the default. Declare the record's string members non-nullable and defend
anyway; a JSON parse can produce a null through a nullable path the compiler does not see.

`Sanitised()` must be idempotent: `s.Sanitised().Sanitised()` equals `s.Sanitised()`.

Do not add a member that formats a style into an SVG attribute string, and do not add anything that
produces a `MarkupString`. D-66's argument is that the whitelist exists precisely so that no
hand-built attribute string is ever needed; providing one here would invite the failure the whitelist
was chosen to make impossible.

Create `src/BlazorCanvas/Shapes/StyleGateway.cs`, `public static class StyleGateway`:

- `public static FigureStyle Parse(string? json)` â€” returns `new FigureStyle().Sanitised()` for null,
  empty, whitespace-only, or unparseable input. Otherwise parses with `JsonDocument.Parse` using
  `JsonDocumentOptions { MaxDepth = 32, AllowTrailingCommas = false, CommentHandling =
  JsonCommentHandling.Disallow }`, wrapped so a `JsonException` returns defaults rather than
  propagating. Never throws for any input.
- `public static FigureStyle Parse(JsonElement json)` â€” the element overload. If `json.ValueKind` is
  not `Object`, return defaults. Read each of the four keys defensively: `stroke` and `fill` are used
  only when present with `ValueKind == JsonValueKind.String`; `stroke_width` and `opacity` only when
  present with `ValueKind == JsonValueKind.Number` and `TryGetDouble` succeeds. Any key that is
  absent or wrong-typed takes its default â€” D-60's standing rule that reads are defensive and a
  missing key takes a default rather than throwing. Unknown keys are ignored, never carried. Always
  return the result of `.Sanitised()`, so no unsanitised `FigureStyle` can leave this class.
- `public static string ToJson(FigureStyle style)` â€” sanitises first, then writes with
  `Utf8JsonWriter` (`Indented = false`) exactly four properties in this fixed order: `stroke`,
  `stroke_width`, `fill`, `opacity`. Fixed order matters because Phase 10 compares stored style text
  and MIGR-03's replay compares migrated rows; a key order that varies would make those comparisons
  ambiguous. Snake-case key names are written as string literals, not derived from a
  `JsonNamingPolicy` â€” a policy change elsewhere in the app must not be able to silently rename a
  stored column's JSON keys.

## Planner Assumptions

- **PA-09 â€” Style JSON keys are `stroke`, `stroke_width`, `fill`, `opacity`, written as literals in a
  fixed order.** D-66 writes `stroke_width` in snake case, so that is the wire form. Literals rather
  than a `JsonNamingPolicy` because a policy is global state that a later phase could change,
  silently renaming keys inside every stored row. Fixed order because Phase 10 and MIGR-03 compare
  stored text.
- **PA-10 â€” `StyleGateway.Parse` never throws; there is no `TryParse` for style.** D-66 says style
  values are *clamped or replaced*, never rejected â€” unlike geometry, which is rejected (VALID-02).
  A style that cannot be read simply becomes the default, which is the current fixed appearance, so
  the worst case is "the figure looks like it does today".
- **PA-11 â€” No SVG-attribute formatting helper is provided.** Deliberate omission: D-66 chose the
  whitelist so that hand-built attribute strings are never necessary. Providing the helper would
  create the exact path the whitelist exists to make unnecessary.
- **PA-12 â€” 09-02 does not edit `GeometryJson.cs`.** 09-01 and 09-02 run in the same wave; that file
  belongs to 09-01. If the shared `Serialise` helper is not yet present, `StyleGateway` implements
  the equivalent `Utf8JsonWriter` output locally. A later consolidation is a one-line change.

The full cross-cutting assumption list (PA-01 â€¦ PA-08) is in `09-01-PLAN.md`.
