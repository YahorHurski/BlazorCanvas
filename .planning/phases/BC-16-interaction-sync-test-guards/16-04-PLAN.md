<objective>
Add the one automated guard this phase is missing: a bUnit render-level smoke test that renders the live star preview through the real component render path and asserts a preview SVG shape appears during a drag, before commit. This complements 16-03's source-text drift guard, which greps Home.razor but never actually renders the preview.

Purpose: TEST-04 requires guards for the milestone's silent failure modes. G-15-1 (Phase 15) rendered nothing for every shape because Home.razor passed PreviewType as a literal instead of an @-bound expression, so FigureShape's Registry.Contains gate rejected it and geometry stayed null. Today only a source-text assertion catches that; nothing renders the preview and asserts SVG is emitted. This plan closes that specific hole at render level.
Output: A pinned test-only bUnit dependency and a new PreviewRenderSmokeTests.cs that renders FigureShape from an active DrawingPreviewSession and asserts a star polygon is emitted at pointermove, with a negative control proving a G-15-1 regression fails the guard. No production change.
</objective>
