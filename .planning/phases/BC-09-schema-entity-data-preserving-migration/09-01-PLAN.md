Purpose: isolate the appearance-preserving per-type maths (verified against the SVG renderer in 09-CONTEXT.md) in one tested place, so no other plan re-derives it and risks a silent figure-altering bug.
Output: `GeometryCodec.cs` + its xUnit test class, built test-first (RED -> GREEN -> REFACTOR).
</objective>

    - Rectangle: Box(x1,y1,x2,y2) -> anchor (x1,y1), geometry {w: x2-x1, h: y2-y1}; DecodeToBox -> Box(x, y, x+w, y+h). Round-trips.
    - Triangle: identical to rectangle (bbox top-left anchor + {w,h}); the renderer reconstructs apex/base from the box.
    - Circle: anchor = centre (x1+r, y1+r) with r=(x2-x1)/2 via CircleEncoding.ToCentreRadius; geometry {r}; DecodeToBox = CircleEncoding.FromCentreRadius(cx, cy, r). Round-trips exactly because the old inscribed square has an even side.
    - Line: anchor (x1,y1), geometry {dx: x2-x1, dy: y2-y1}; dy may be negative; DecodeToBox -> Box(x, y, x+dx, y+dy). A down-right diagonal (dx>0, dy>0) and an up-right diagonal (dx>0, dy<0) both round-trip without the endpoint pair swapping.
    - All outputs integer-valued; the geometry serialises to compact JSON with lowercase keys exactly matching the D-59 shapes.
  </behavior>
