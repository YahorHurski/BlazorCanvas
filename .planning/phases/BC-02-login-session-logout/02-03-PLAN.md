<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser -> /login EditForm POST | Untrusted credentials + form fields cross into the login handler |
| browser -> logout `<form>` POST | Potentially cross-site POST reaches the sign-out endpoint |
| cookie claim -> canvas circuit | `user_id` claim read by the interactive shell to scope later figure queries |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-301 | Spoofing / Tampering | `/login` EditForm POST (CSRF) | high | mitigate | `EditForm` auto-adds the antiforgery token; `UseAntiforgery()` (02-02) validates it |
| T-02-302 | Spoofing / Tampering | Logout `<form>` POST (CSRF) | high | mitigate | Plain form gets NO auto token -> manual hidden input via `AntiforgeryStateProvider.GetAntiforgeryToken()`; endpoint validates (02-02) |
| T-02-303 | Tampering | open redirect via `ReturnUrl` | medium | mitigate | `ReturnUrl` value is NEVER used for navigation - presence only drives the banner; success always `NavigateTo("/")` (RESEARCH OQ2) |
| T-02-304 | Information Disclosure | username enumeration via distinct error copy | low | accept | D-17 / UI-SPEC REQUIRE distinct "Wrong password." vs silent account creation - explicitly ACCEPTED (no real security boundary, D-08); do not unify messages |
| T-02-305 | Tampering / DoS | first-seen-username race (concurrent INSERT) | medium | mitigate | Catch `DbUpdateException` (Npgsql UniqueViolation 23505), detach, re-query, fall through to password compare (RESEARCH Pitfall 3) |
| T-02-306 | Elevation of Privilege | `user_id` claim integrity | medium | mitigate | Claim rides inside the Data-Protection-encrypted, tamper-evident auth cookie; read via cascading `AuthenticationState`, no client-writable token |
| T-02-307 | Information Disclosure | plaintext password entered/compared | low | accept | D-08 LOCKED - deliberate throwaway project; no hashing UI, no "show password" toggle (UI-SPEC); recorded as ACCEPTED, not a finding to fix |
| T-02-3SC | Tampering | npm/pip/cargo installs | low | accept | No package-manager installs in this phase (all APIs in-box) - supply-chain gate N/A |
</threat_model>
