<objective>
Remove the canvas-edge clamp from the geometry core and re-express the degenerate-draw guard on the
anchor+geometry primitives. After this plan a draw gesture reproduces exactly what the pointer did ‚Äî
no coordinate is bent toward the canvas ‚Äî and the only thing that can still refuse a draw is the
per-type zero-extent guard, reading the same integers that will later be serialised into `geometry`.

This is the foundation plan: `Movement.ClampMove` (the *drag* clamp) is deliberately left alone here
because `Home.razor` still calls it; 10-02 deletes it together with the drag rework.

Purpose: STOR-04's "a figure may now be drawn past the canvas edge" and STOR-03's "well-formedness is
guaranteed by server-side code, not the database" both start in these four files.
Output: unclamped `DrawGesture`, a two-member `CircleEncoding`, a primitives-reading `MinSizeGuard`, a
non-finite-safe `CanvasCoordinates`, and the four test files that pin them.
</objective>

planner decisions, not user decisions, and each is reversible:

- **PA-1 ‚Äî `Box` is retained, and its role is narrowed.** `Box` stops being a storage shape and becomes
  (a) the transient extent a draw gesture produces and (b) the decoded extent the migration codec
  speaks. Renaming it to something like `Extent` was considered and rejected: it would churn
  `GeometryCodec`, the Phase-9 migration round-trip proof, and every geometry test, for a name.
  Its doc comment is corrected instead.
- **PA-7 ‚Äî `MinSizeGuard`'s circle arm drops the square-ness and even-side terms.** Those existed
  solely to mirror the `circle_is_a_circle` CHECK that D-59 deleted. The replacement rule reads the
  integer radius that `{r}` will actually carry, which also fixes a latent hole: a width-1 box passes a
  naive `width > 0` rule but serialises as `{"r":0}` ‚Äî an invisible, unselectable poison row.
- **PA-5 ‚Äî `CanvasCoordinates.FromPage` gains a non-finite guard.** This is NOT a reinstated canvas
  clamp; it bounds the value to the `int` domain the `x`/`y` columns already are. It exists because the
  removed clamp was, incidentally, the only thing that previously turned a crafted `NaN`/`Infinity`
  circuit payload into a sane integer (T-10-01).
</planner_assumptions>
