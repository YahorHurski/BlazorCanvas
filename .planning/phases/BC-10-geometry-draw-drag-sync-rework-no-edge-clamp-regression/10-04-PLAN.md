<objective>
Move the renderer onto the storage model. `FigureShape` and `SelectionTrace` stop taking a bounding box
and start taking the figure's anchor and its `geometry` JSON, deriving SVG coordinates through one
shared `ShapeRender` helper. `Home.razor` drops the box helpers that have bridged the two models since
10-02, and the live preview renders from the same anchor+geometry pair the commit will persist.

The bar is pixel-identity: this milestone adds no user-facing change, so every coordinate the new path
emits must equal the coordinate the old path emitted for the same figure. `ShapeRenderTests` asserts
exactly that equivalence.

Purpose: ROADMAP success criterion 1 ‚Äî "renders from its anchor (x,y) + geometry instead of the retired
bounding box, and looks identical to before".
Output: `ShapeRender`, two reworked components, an anchor-based `Home.razor` render, and the
appearance-preservation tests.
</objective>

<planner_assumptions>
- **PA-2 ‚Äî the components take the raw `geometry` JSON string, not a typed geometry union.** No
  CONTEXT.md pinned this. A C# discriminated union over four fixed, never-growing shapes would be
  ceremony; the raw string is also exactly what the `jsonb` column and the `draw` broadcast carry, so
  there is one representation from database to pixel. Each component decodes once per parameter change
  rather than once per markup expression.
- **PA-11 ‚Äî `ShapeRender` is a new class rather than more members on `GeometryCodec`.** The codec's job
  is the storage boundary and the migration proof depends on it; the render helper's job is SVG
  coordinates and a locale-safe points list. Keeping them apart keeps the Phase-9 artefact stable.
</planner_assumptions>

  <action>
    Create a static `ShapeRender` in the `BlazorCanvas.Geometry` namespace with four members:
    a rectangle/triangle size accessor returning the stored width and height as a named integer pair;
    a circle radius accessor returning the stored radius as an integer; a line delta accessor returning
    the stored signed dx and dy as a named integer pair; and a triangle points formatter taking the
    anchor and the width and height and returning the SVG `points` attribute value.

    The three accessors deserialise the geometry JSON, reading exactly the member names `w`, `h`, `r`,
    `dx`, `dy` that `GeometryCodec` writes. Do not re-derive any of those names ‚Äî read them off the
    codec so a rename in one place cannot silently diverge.

    The points formatter must reproduce the current output exactly: apex x is the anchor x plus half the
    width as a FRACTIONAL value, apex y is the anchor y, base-left is the anchor with y plus the height,
    base-right is the anchor x plus the width with y plus the height. Integer division is forbidden ‚Äî it
    makes odd-width triangles non-isosceles and breaks D-21's promise. Format the whole string through
    InvariantCulture: on a comma-decimal server locale a half-pixel apex would otherwise emit a comma
    and the browser would reparse the points list as different coordinates.

    Write `ShapeRenderTests` as a pure unit class. Cover:
    - Each accessor against the exact JSON literals the codec emits for each type.
    - The points formatter for an even width and for an odd width, asserting the half-pixel apex on the
      odd case.
    - A locale test: set the current culture to a comma-decimal culture such as `de-DE` for the duration
      of the assertion and confirm the emitted points string still uses a period as the decimal
      separator. Restore the culture afterwards.
    - The appearance-preservation cross-check, for all four types: start from a `Box`, encode it with
      `GeometryCodec.Encode`, then assert the coordinates derived through `ShapeRender` equal the
      coordinates the bounding-box renderer produced ‚Äî the line's two endpoints equal the box's two
      corners; the rectangle's x, y, width and height equal the box's X1, Y1, Width and Height; the
      circle's centre and radius equal `CircleEncoding.ToCentreRadius` of the box; and the triangle's
      points string equals the string the old formatter built from the same box. Include a
      down-and-right diagonal line and an up-and-right diagonal line so a flipped line cannot pass.
  </action>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| stored `geometry` ‚Üí SVG markup | The JSON originates server-side from `GeometryCodec` and is rendered into SVG attributes by Blazor, which encodes attribute values; no client-supplied geometry exists on this path. |
| server culture ‚Üí SVG numeric formatting | `CultureInfo.CurrentCulture` on a Blazor Server circuit is the host OS culture unless request localization is configured. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-10-13 | Tampering | `ShapeRender` triangle points formatter | medium | mitigate | A comma-decimal host culture would emit a half-pixel apex as `4,5`, and the browser would silently reparse the points list as an extra coordinate pair ‚Äî a wrong shape with no error anywhere. Mitigated by formatting through InvariantCulture and by a unit test that sets `CultureInfo.CurrentCulture` to `de-DE` and asserts a period separator. |
| T-10-14 | Tampering | anchor+geometry render path | high | mitigate | A wrong per-type derivation silently alters how every existing figure looks, and users cannot re-draw what they lost. | T-10-15 | Information disclosure | geometry string in SVG attributes | low | accept | The rendered values are integers the server itself computed and are already visible as the drawing. Blazor attribute binding encodes values; no raw string is interpolated into markup. |
| T-10-SC | Tampering | package installs | low | accept | No NuGet, npm, pip or cargo package is added in this phase. The Package Legitimacy Gate does not apply. |
