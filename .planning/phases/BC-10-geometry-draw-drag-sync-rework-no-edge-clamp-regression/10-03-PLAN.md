<objective>
Rework the D-53 broadcast contract onto anchor + geometry, with every semantic unchanged. A `draw`
now carries the anchor, the type and the geometry JSON; a `move` and a `rollback` carry the anchor
alone; a `delete` still carries only the id. The receiver rules that make those payloads safe ‚Äî draw is
the only kind that may create, move and rollback are update-only, delete is idempotent ‚Äî move out of
`Home.razor` into a small UI-free `SyncReceiver` so they can finally be unit-tested instead of only
being observed on two monitors.

Purpose: SYNC-02. The payload changes; the semantics must not.
Output: the new `SyncMessage` record, a tested `SyncReceiver`, and a `Home.razor` that publishes and
consumes the new shape.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| circuit ‚Üí CanvasSyncNotifier ‚Üí circuit | An in-process singleton fans messages out to every subscriber bucket keyed by `user_id`; the bucket key is the whole cross-user isolation boundary. |
| notifier callback ‚Üí component | Callbacks run on the publisher's circuit and are marshalled back with `InvokeAsync` before touching component state. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-10-08 | Information disclosure | `CanvasSyncNotifier.Publish` | high | mitigate | The `draw` payload now carries `geometry` as well as an anchor, so a mis-keyed publish would leak shape data and not merely a position. Mitigated by leaving the per-`user_id` bucket keying untouched and keeping `Publish_ForDifferentUser_DoesNotInvokeHandler` and the multi-subscriber test in `CanvasSyncNotifierTests` as the standing proof. |
| T-10-09 | Tampering | `SyncReceiver` move and rollback arms | high | mitigate | If either arm inserted on an unknown id it would resurrect a figure the user deleted ‚Äî the exact D-40 bug. Mitigated by an explicit early return on an absent id plus two unit tests asserting the list is unchanged and nothing is created. |
| T-10-10 | Spoofing | `SyncMessage.Sender` | low | accept | The per-circuit guid exists solely for D-11's echo filter; it is not an authentication token and carries no authority. The notifier is in-process and single-trust-domain, and the bucket key ‚Äî not the sender ‚Äî is what scopes delivery. |
| T-10-11 | Denial of service | drag glide broadcast volume | medium | mitigate | An unthrottled glide would publish on every pointer-move to every subscriber of the user. Mitigated by leaving D-47's 50 ms trailing-edge throttle exactly as it is; the payload rework must not disturb the tick bookkeeping. |
| T-10-12 | Tampering | geometry string on the wire | low | accept | The geometry travels as the same JSON the server itself wrote through `GeometryCodec`; it originates server-side, is never round-tripped through the client, and the notifier is in-process. No client-supplied geometry exists anywhere on this path ‚Äî the draw path transmits only a tool name and two points. |
| T-10-SC | Tampering | package installs | low | accept | No NuGet, npm, pip or cargo package is added in this phase. The Package Legitimacy Gate does not apply. |
</threat_model>
