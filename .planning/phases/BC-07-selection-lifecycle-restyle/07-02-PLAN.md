    and the SEL-02 restyle (a topmost ~1px blue `#1D4ED8` + white dashed trace on the figure's own
    outline via the new `SelectionTrace` component, replacing the v1.0 red 2px outline removed from
    FigureShape).
  </what-built>
  <how-to-verify>
    Run the app (`dotnet run --project src/BlazorCanvas`) and log in. Then check each item — each maps
    to a ROADMAP Phase 7 success criterion:

    1. **Draw keeps tool armed + auto-selects (criterion 1):** arm the Rectangle tool and draw a
       rectangle. Expected: the Rectangle toolbar button STAYS highlighted (armed, blue), AND the
       just-drawn rectangle immediately shows the blue+white dashed trace (it is selected). Draw a
       second rectangle without re-arming — it should draw and become the new selection.

    2. **At most one selected (criterion 2):** with several figures on the canvas, arm the Pointer
       tool and click figures one by one. Expected: exactly one figure shows the trace at any time —
       selecting another moves the trace; there are never two traces.

    3. **Deselect triggers + Delete still works (criterion 3):** select a figure, then (a) press empty
       canvas with the Pointer armed → trace clears; select again and (b) arm any shape tool → trace
       clears; select again and (c) press the Pointer/logout/other toolbar button → trace clears; then
       select again and press **Delete** → the figure is deleted (Delete did NOT merely deselect — it
       acted on the selection).

    4. **Topmost blue+white dashed trace on the figure's own outline, visible behind a larger figure
       (criterion 4):** select a small figure, then draw a LARGER figure on top of it (its white fill
       covers the small one). Expected: the small figure's dashed trace remains visible ON TOP of the
       larger figure (topmost, `pointer-events:none`). Confirm the trace follows the figure's own shape
       (a circle's trace is a circle, a triangle's is a triangle) — NOT a rectangular bounding box.
       Drag the selected figure and confirm the trace tracks it live.

    5. **No red selection outline anywhere (criterion 5):** select each of the four shape types in
       turn. Expected: no figure's own outline ever turns red on selection (the outline stays black;
       only the separate blue+white trace appears). Note: the Delete button turning red on hover is
       intentional and expected — that is not a figure selection outline.

    6. **Two-tab concurrency edge (Phase 5 sync):** open the app in two tabs of the SAME account. In
       tab A, select a figure. In tab B, select and Delete that SAME figure. Expected: in tab A the
       figure disappears AND its selection trace disappears with it (no dashed trace left hovering over
       empty canvas).
  </how-to-verify>
