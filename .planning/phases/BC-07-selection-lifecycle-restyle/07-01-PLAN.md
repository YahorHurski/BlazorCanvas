<objective>
Implement the SEL-01 selection **lifecycle** and the SEL-02 selection **restyle** — the two changes
that make figure selection predictable and calm. Both are fixed by locked decisions **D-30**
(pointer vs draw modes), **D-31** (v1.1 appearance + lifecycle) and **D-58** (the pinned constants);
nothing here is an open design question.

SEL-01: after a draw the armed tool STAYS armed and the just-drawn figure is auto-selected; at most
one figure is ever selected; the selection clears on canvas-outside-press, on arming any tool, and on
pressing any toolbar button EXCEPT Delete (Delete stops propagation so it still acts on the
selection). SEL-02: the selected figure is indicated by a ~1px blue (`#1D4ED8`) + white dashed trace
riding the figure's OWN outline, rendered as the topmost `<svg>` child with `pointer-events:none` so
it stays visible even behind a later, larger figure — replacing the v1.0 red 2px outline, which is
removed from FigureShape.

Explicitly OUT of scope (do not touch): the 1472x828 canvas bounds and figure positions (Phase 6 —
do not regress), the geometry/clamp maths, the DB schema, sync message contract, and auth. No
database migration, no new figure type, no new package, and **no hand-authored JavaScript** (ARCH-01
permits it from v1.1 but this milestone writes none — all interaction stays Blazor Server pointer/
click handlers). Selection is local UI state only — never persisted, never broadcast (D-31/D-11).

Purpose: Deliver the fixed, one-at-a-time selection lifecycle and the topmost blue+white dashed trace
so selecting reads as a calm, non-destructive state distinct from the red Delete affordance.
Output: Updated `Home.razor`, `Toolbar.razor`, `FigureShape.razor`, and a new `SelectionTrace.razor`.
</objective>
