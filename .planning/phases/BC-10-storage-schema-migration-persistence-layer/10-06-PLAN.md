intentional migration failure.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| legacy public rows → v11 migration | Existing persisted data is converted into a new schema and must not leave a partial database state when conversion rejects a row. |
| migration transaction → PostgreSQL DDL/catalog | Transactional DDL and lookup-table writes must have the same rollback boundary as canvas and figure writes. |

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-10-06-01 | Tampering | `V11DataMigration.RunAsync` | high | mitigate | Begin one transaction before v11 DDL and pass it to schema, seed, canvas, and figure commands; commit only after all conversion checks pass. |
| T-10-06-02 | Tampering | `V11Schema` command construction | medium | mitigate | Bind the supplied `NpgsqlTransaction` to DDL and parameterised seed commands, retaining schema-qualified SQL and conflict-safe seed handling. |
| T-10-06-03 | Information disclosure | invalid-row test failure | low | mitigate | Preserve the existing assertion that migration exceptions do not reveal legacy coordinates. |
</threat_model>
